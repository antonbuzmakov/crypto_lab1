# Лабораторная работа. ЗАЩИЩЕННЫЙ ЧАТ С ИСПОЛЬЗОВАНИЕМ RSA И AES

## БЫСТРЫЙ СТАРТ

Все, что нужно для того чтобы запустить программу. Пункты расписаны ниже.
- Установка зависимостей
- Выбор подключения
- Запуск чата

## УСТАНОВКА ЗАВИСИМОСТЕЙ

`pip install cryptography`

## ДЛЯ РАБОТЫ ЧЕРЕЗ RADMIN VPN

- Установите Radmin VPN
- Боб создает сеть, Алиса подключается
- Боб сообщает свой IP-адрес Radmin (26.x.x.x)

## ДЛЯ РАБОТЫ ЧЕРЕЗ localhost

- Запустите программу в двух разных терминалах (Алиса и Боб)
- Боб создает сеть, Алиса подключается
- Алиса подключается используя 127.0.0.1

## ЗАПУСК ЧАТА

**Боб (сервер, ожидает подключение):**  
`python secure_chat.py` → имя bob → режим 1 → порт 12345 (Enter)

**Алиса (клиент, подключается):**  
`python secure_chat.py` → имя alice → режим 2 → IP Боба (Radmin или локальный) → порт 12345

## ПРОТОКОЛ

- Обмен открытыми ключами RSA
- Двусторонняя аутентификация (подписи RSA)
- Передача сессионного ключа AES-256
- Защищенное общение (шифрование AES-GCM)

## КОМАНДЫ В ЧАТЕ

`exit` — выход из чата

## ГЕНЕРАЦИЯ КЛЮЧЕЙ

На каждом участнике: `python generate_keys.py` → введите имя (alice или bob).  
Создаются файлы:

- `alice_private.pem` / `bob_private.pem` (секретные!)
- `alice_public.pem` / `bob_public.pem` (для обмена)

## ОБМЕН КЛЮЧАМИ

1. Алиса отправляет `alice_public.pem` Бобу → Боб сохраняет как `peer_public.pem`
2. Боб отправляет `bob_public.pem` Алисе → Алиса сохраняет как `peer_public.pem`

## СОЗДАННЫЕ ФАЙЛЫ

- `alice_private.pem` / `bob_private.pem` — секретные ключи
- `alice_public.pem` / `bob_public.pem` — публичные ключи
- `peer_public.pem` — полученный ключ собеседника
